# PRD: Flow Planner

## 제품 개요

**반복되는 Workflow Task를 하루 단위로 실체화**하고, 사용자가 **눈에 보이는 성취를 통해 안정감과 몰입을 얻도록 돕는 생산성 도구**.

### 핵심 철학

- Todo 앱이 아닌, **'하루의 흐름을 눈에 보이게 쌓아주는 시스템'**
- Task는 **덩어리**로, Flow는 **길**로, 캘린더는 **땅**으로 작동
- 완료는 사라짐이 아니라 **누적**

### 타겟 사용자

- 해야 할 일이 **직관적으로 보일 때** 성취감을 느끼는 사람
- 여러 단계(Sub-Flow)를 가진 작업을 반복 수행하는 사람

---

## 핵심 문제 정의

### 사용자 문제

- 반복 Task가 매일 머릿속에만 존재해 불안함
- 작업이 여러 단계로 구성되어 있어 진행 상황이 추상적
- 완료해도 '사라지는 Todo' 때문에 성취감이 축적되지 않음

### 해결 목표

- 하루 단위로 Task를 **명확한 실체**로 생성
- Task 내부의 Flow를 **진행 중인 길**처럼 시각화
- 완료 결과가 **캘린더에 누적되어 쌓이는 구조** 제공

---

## 핵심 컨셉

### Task

- 의미 있는 작업 단위
- 하루에 캘린더에 1번만 나타남
- 내부적으로 여러 Sub-Flow 단계를 가짐

### Sub-Flow (Flow Steps)

- Task 완료를 위해 거쳐야 하는 고정된 단계들
- 순서가 있으며 단독으로 존재하지 않음
- 반복 Task의 경우 **템플릿으로 재사용**

### Task Template vs Task Instance

- **Task Template**: 반복 규칙과 Flow를 정의하는 설계도
- **Task Instance**: 특정 날짜에 실제로 생성된 작업 실체

---

## 사용자 흐름

### 하루 시작

1. 앱을 열고 `[오늘 Task 생성]` 버튼 클릭
2. 반복 Task 템플릿 목록이 체크리스트로 표시
3. 오늘 수행할 Task만 선택 후 생성
4. 선택된 Task들이 **오늘의 Task Instance**로 생성

### 작업 수행

- FlowBoard에 오늘 생성된 Task만 표시
- 각 Task 카드 내부에서 Sub-Flow 단계 체크
- 진행 바 및 시각적 피드백 즉시 반영

### 완료 및 성취

- Task 완료 시 카드가 사라지지 않고 **완료 영역으로 이동**
- 캘린더 날짜 칸에 완료된 Task 수만큼 시각적 채움

---

## UX 설계 원칙 (토 기운 보충)

1. 오늘 단위로 실체화
2. 눈에 보이는 덩어리
3. 완료는 사라짐이 아니라 누적
4. 캘린더는 계획 도구가 아닌 **성과 증거판**

---

## MVP 범위 (Out of Scope)

- 다중 사용자 OAuth
- 고급 AI 추천 로직
- 과거/미래 날짜 대량 편집

## 성공 지표

- 하루 Task 생성 버튼 클릭률
- 하루 Task 완료 개수
- 연속 완료된 날짜 스트릭
- 사용자가 느끼는 '오늘이 채워졌다'는 주관적 만족도
